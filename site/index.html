How it works

Hook → tiny server → the board.

Each tool's native hook reports state changes — no change to your workflow. A small HTTP server on your computer relays them; any screen holds a live stream and shows the change in well under a second.

01 · detect

Native hooks

Claude Code & Codex fire lifecycle hooks. andon install claude wires them for you (with a backup).

02 · relay

Your computer

A zero-dependency Node server receives the events. macOS · Linux · Windows. Nothing leaves your machine.

03 · display

Any screen

iPad, phone, or browser — a signal bar across the top is the tower light; the most-urgent row floats up and pulses.

Why it's different

Built like a tool, not a SaaS.

multi-agent

Several at once

Zero dependencies

Pure Node.js standard library. No build chain, no account, no hardware — `npx` and you're live.

private

No telemetry, no analytics, no phone-home. It only ever holds high-level status — never your code or logs.

Board from anywhere

Phone push, your way.

Local-first and free to self-host forever. To reach the board — and get push — from anywhere, route through a relay that stores ciphertext only and can't read your agents' content.

☁️ Managed relay launching

The easy way — no server

Zero setup, no Tailscale, no HTTPS to configure. Opt in and get phone push from anywhere.

$ andon hosted setup https://relay.agentandon.com
⌘ Self-host

Run your own relay

Same open-source code. Or expose the local board to just your devices with Tailscale.

$ andon relay · andon verify <url>
🔒 Verifiable, not just trusted. The relay is end-to-end encrypted and reproducible from source — andon verify confirms it serves the exact open-source code.
