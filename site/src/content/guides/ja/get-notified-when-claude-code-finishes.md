Claude Code にタスクを任せ、別の作業へ切り替える —— そして…待ちます。終わったでしょうか？ 「はい」の一言を待ったまま、プロンプトの前で止まっているのでしょうか？ 確認しようとタブを切り替えると、4 分前にとっくに終わっていた —— あるいはもっと悪いことに、ずっと止まったままだった。これを複数のエージェント分くり返せば、一日はターミナルのお守りで終わってしまいます。

**Agent Andon** はこれを解決します。コーディングエージェントを見張り、どれかが**完了した**・**あなたの入力を必要とした**・**止まった**瞬間に通知します —— どの画面でも開けるボード上で、デスクトップやスマホの通知（任意）と一緒に。

## Agent Andon をインストールする

```
npm i -g agent-andon
```

依存ゼロの CLI で、ローカルで動きます —— アカウント不要、テレメトリーなし。

## エージェントのフックを繋ぐ

Andon は各ツールの**ネイティブなライフサイクルフック**を読み取ります —— エージェントをラップしたりプロキシしたりはしません。

```
andon install claude
```

これだけです。Claude Code は、いつもの作業を一切変えることなく、状態変化（作業中 → 確認待ち → 完了 → 停止）を報告するようになります。OpenAI Codex も動かしていますか？ `andon install codex` で同じことができます。

## 各状態の意味

- **作業中** —— エージェントは作業中。あなたがすることはありません。
- **確認待ち** —— プロンプト、許可、または判断を待っています。いち早く気づきたいのはこれです。
- **完了** —— エージェントは自分のターンを終え、あなたにバトンを返しました。
- **停止** —— エラーになったか、動かなくなりました。

## どの画面でもボードを開く

```
andon serve
```

表示された URL を、任意のブラウザ・スマホ・壁掛けの iPad で開きます。各セッションは 1 行として表示され、**確認待ち**のものが上に浮かび上がります —— だから、ひと目でどこを見ればいいかわかります。

## デスクトップとスマホで通知を受け取る

**デスクトップのバナー通知**はデフォルトでオンです。**メニューバーのサマリー**はあとひと繋ぎ —— Andon は `/menubar` でプレーンテキストのステータスを配信するので、そこへ SwiftBar・xbar・Waybar を向けるだけです。

**どこからでもスマホにプッシュ通知**を受け取るには —— マシンから離れていても —— **コンテンツブラインド中継**を接続します。これは、あなたのプロジェクト名やメッセージを読み取ることなく通知を転送します。次のコマンドで Andon を中継に向けます：

```
andon hosted setup <relay-url>
```

自分で中継を動かすことも、マネージドの中継（まもなく公開）を使うこともできます。デスクトップとメニューバーの詳細は [通知](/ja/docs/notifications/) を、中継については [ホスト型 Andon](/ja/docs/hosted/) を参照してください。

## Codex でも使えます

ここまでのすべては **OpenAI Codex** にもそのまま当てはまります —— `andon install codex`、同じボード、同じ通知。Claude Code と Codex のセッションを並べて見張れます。

---

これがすべての流れです。インストールし、フックを繋ぎ、ボードを開き、通知をオンにする。エージェントが完了したり、あなたの手を必要としたりすることが、10 分遅れで気づくものではなく、通知になります。
