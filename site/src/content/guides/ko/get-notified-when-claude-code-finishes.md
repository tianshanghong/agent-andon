Claude Code에게 어떤 작업을 맡기고, 다른 일로 탭을 옮겼다가… 기다립니다. 끝났을까? 여러분의 "예"를 기다리며 프롬프트에서 멈춰 있는 건 아닐까? 확인하려고 alt-tab으로 다시 돌아가 보면 4분 전에 이미 끝나 있거나 — 더 나쁘게는 그동안 내내 멈춰 있던 경우도 있습니다. 여기에 에이전트 여러 개를 곱하면 하루가 터미널을 돌보는 일로 변해 버립니다.

**Agent Andon**은 이 문제를 해결합니다. 여러분의 코딩 에이전트를 지켜보다가 하나가 **끝나거나**, **입력이 필요하거나**, **막히는** 순간 바로 알려 줍니다 — 어떤 화면에서나 열 수 있는 보드에서, 선택적인 데스크톱·휴대폰 알림과 함께.

## Agent Andon 설치하기

```
npm i -g agent-andon
```

로컬에서 실행되는, 의존성이 없는 CLI입니다 — 계정도, 텔레메트리도 없습니다.

## 에이전트의 훅 연결하기

Andon은 각 도구의 **네이티브 라이프사이클 훅**을 읽습니다 — 여러분의 에이전트를 감싸거나 프록시하지 않습니다.

```
andon install claude
```

이게 전부입니다. 이제 Claude Code는 작업 방식을 전혀 바꾸지 않고도 상태 변화(작업 중 → 확인 필요 → 완료 → 막힘)를 보고합니다. OpenAI Codex도 쓰시나요? `andon install codex`가 똑같이 처리해 줍니다.

## 각 상태의 의미

- **작업 중** — 에이전트가 일하는 중이며, 여러분이 할 일은 없습니다.
- **확인 필요** — 프롬프트나 권한, 또는 결정을 기다리는 중입니다. 빠르게 잡아낼 가치가 있는 바로 그 상태입니다.
- **완료** — 에이전트가 자기 차례를 마치고 여러분에게 넘겼습니다.
- **막힘** — 오류가 났거나 멈춰 섰습니다.

## 어떤 화면에서든 보드 열기

```
andon serve
```

출력된 URL을 아무 브라우저, 휴대폰, 또는 벽에 걸어 둔 iPad에서 여세요. 모든 세션이 한 줄로 표시되고, **확인 필요**인 세션은 맨 위로 떠오릅니다 — 그래서 한눈에 어디를 봐야 할지 알 수 있습니다.

## 데스크톱·휴대폰 알림 받기

**데스크톱 배너**는 기본적으로 켜져 있습니다. **메뉴 막대 요약**은 연결 한 번이면 됩니다 — Andon은 `/menubar`에서 일반 텍스트 상태를 제공하며, 여기에 SwiftBar, xbar, 또는 Waybar를 연결하면 됩니다.

머신에서 멀리 떨어져 있어도 **어디서나 휴대폰 푸시**를 받으려면 **콘텐츠 블라인드 릴레이**를 연결하세요. 이 릴레이는 여러분의 프로젝트 이름이나 메시지를 읽지 못한 채 알림을 전달합니다. 다음 명령으로 Andon을 릴레이에 연결하세요:

```
andon hosted setup <relay-url>
```

직접 릴레이를 운영하거나, 관리형 릴레이(곧 출시)를 사용할 수 있습니다. 데스크톱과 메뉴 막대에 대한 자세한 내용은 [알림](/ko/docs/notifications/)을, 릴레이에 대해서는 [Hosted Andon](/ko/docs/hosted/)을 참고하세요.

## Codex에서도 작동합니다

위의 모든 내용은 **OpenAI Codex**에도 그대로 적용됩니다 — `andon install codex`, 같은 보드, 같은 알림. Claude Code와 Codex 세션을 나란히 지켜보세요.

---

이것이 전체 흐름입니다. 설치하고, 훅을 연결하고, 보드를 열고, 알림을 켜기. 에이전트가 일을 끝내거나 여러분의 손이 필요해지면 그것이 알림이 됩니다 — 10분 늦게 발견하는 일이 아니라.
