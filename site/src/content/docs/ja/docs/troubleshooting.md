## トラブルシューティング

- **ボードを表示するデバイスがページを開けない** —— 同じ Wi-Fi につながっていますか？ `https` ではなく
  `http` ですか？ コンピューターのファイアウォールが受信接続を許可していますか（macOS：システム設定 →
  ネットワーク → ファイアウォール）？ IP は正しくコピーできていますか（起動時に表示され、`andon doctor` でも再表示されます）？
- **Claude のフックが何も起こさない** —— 一度 `claude --debug` を実行してフックのエラーが出ていないか確認します。
  `andon install claude` を実行し直し、`andon doctor` で確認します。
- **Codex のタイルが表示されない／変化しない** —— Codex 内で一度 `/hooks` を実行してフックを信頼します
  （Codex は信頼されていないフックをスキップします）。`andon doctor` で接続を確認できます。
- **「作業中」のタイルが固まっている** —— プロセスが終了イベントを送る前に死んでしまった可能性があります。
  6 時間後に自動的にクリアされます。Codex の場合は、そのプロジェクトのディレクトリで `andon post gone codex` を実行すれば今すぐクリアできます。
- **ボードで通知音が鳴らない** —— 一度 **通知を有効化** をタップします（ブラウザはタップするまで音声をミュートします）。
  スマホでは、プッシュ通知のためにボードが **HTTPS** で配信されている必要があります（[running.md](/ja/docs/running/) を参照）。

## FAQ

**Claude Code が完了したり承認を求めたりしたときに通知を受け取るには？**
`andon serve`（デスクトップ通知はデフォルトでオン）と `andon install claude` を実行します。セッションがあなたの対応を
必要とした瞬間、または完了した瞬間にデスクトップのバナーが表示され、さらにどのデバイスでもライブのボードを確認できます。

**複数の Claude Code / Codex セッションを同時に見守れますか？**
はい —— それこそが狙いです。各セッションがそれぞれ 1 行になり、あなたの対応を必要とするものが最上部に浮かび上がります。

**OpenAI Codex でも使えますか？**
はい。`andon install codex` が Codex のライフサイクルフックを接続します（一度 `/hooks` を実行して信頼してください）。

**本当に iPad が必要ですか？**
いいえ。ボードはただの Web ページです —— スマホ、タブレット、ブラウザのどれでも開けます。使っていない iPad は、
常時表示の壁掛けディスプレイとして見栄えがよいというだけのことです。デスクトップのバナーやメニューバーのサマリーも利用できます。

**自分のコードやデータがどこかに送られますか？**
いいえ —— デフォルトでは、あなたのエージェントに関する情報は一切マシンの外に出ません。Andon は完全にセルフホストで、
アカウントも、テレメトリも、アナリティクスも、「phone home」（外部への自動通信）もありません。保持するのは高レベルの
ステータス（状態、プロジェクト名、1 行のメッセージ）だけで、あなたのコードやログ、シークレットを保持することはありません。

率直なただし書きが 3 つあります：(1) ボードは、自分でセルフホストしない限り、Web フォントを Google Fonts から読み込みます
—— このリクエストにエージェントのデータは一切含まれず、ブラウザによる通常のフォント取得にすぎません。(2) オプション機能
（スマホへのプッシュ通知と、ホスト型中継）は **完全にオプトイン式** で、それぞれ何がマシンの外に出るのかを正確に明示します
—— ホスト型中継は、*それ自身* でさえあなたのエージェントのメッセージを読めないように設計されています。これらがこの
ローカルファーストの既定を変えることは決してありません。(3) このドキュメントサイト自体（Andon というツールではなく）は、
訪問数を数えるために Cloudflare の Cookie を使わない Web Analytics を利用しています —— Cookie なし、クロスサイト
トラッキングなし、エージェントのデータなし。あなたが動かすボードや中継がこれを読み込むことはありません。
