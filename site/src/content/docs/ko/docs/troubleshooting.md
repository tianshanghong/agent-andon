## 문제 해결

- **보드 기기에서 페이지를 열 수 없습니다** — 같은 Wi-Fi인가요? `https`가 아니라 `http`인가요? 컴퓨터
  방화벽이 들어오는 연결을 허용하나요(macOS: 시스템 설정 → 네트워크 → 방화벽)? IP를 정확히 복사했나요
  (시작할 때 출력되며, `andon doctor`가 다시 출력합니다)?
- **Claude 훅이 아무 일도 하지 않습니다** — `claude --debug`를 한 번 실행해 훅 오류가 있는지 지켜보세요;
  `andon install claude`를 다시 실행하세요; `andon doctor`로 확인하세요.
- **Codex 카드가 전혀 나타나지 않거나 / 전혀 바뀌지 않습니다** — Codex 안에서 `/hooks`를 한 번 실행해
  훅을 신뢰하세요(Codex는 신뢰하지 않는 훅을 건너뜁니다); `andon doctor`가 연결 상태를 확인해 줍니다.
- **"작업 중" 카드가 멈춰 있습니다** — 프로세스가 종료 이벤트를 보내기 전에 죽었을 가능성이 큽니다.
  6시간 뒤 자동으로 사라집니다; Codex의 경우 해당 프로젝트 디렉터리에서 `andon post gone codex`를 실행하면 지금 바로 제거됩니다.
- **보드에서 소리(차임)가 나지 않습니다** — **알림 켜기**를 한 번 탭하세요(그 전까지 브라우저는 오디오를 음소거합니다);
  휴대폰에서는 푸시를 받으려면 보드가 **HTTPS**로 제공되어야 합니다([running.md](/ko/docs/running/) 참고).

## FAQ

**Claude Code가 작업을 끝내거나 승인을 요청할 때 어떻게 알림을 받나요?**
`andon serve`(데스크톱 알림은 기본적으로 켜져 있습니다)와 `andon install claude`를 실행하세요. 세션이 여러분의
손이 필요하거나 작업을 끝내는 순간 데스크톱 배너가 뜨고, 더불어 어떤 기기에서나 실시간 보드도 볼 수 있습니다.

**여러 Claude Code / Codex 세션을 한 번에 모니터링할 수 있나요?**
네 — 그게 바로 핵심입니다. 모든 세션은 각자 한 줄을 차지하며, 여러분의 손이 필요한 것은 무엇이든 맨 위로 떠오릅니다.

**OpenAI Codex와도 동작하나요?**
네. `andon install codex`가 Codex의 라이프사이클 훅을 연결합니다(`/hooks`를 한 번 실행해 신뢰하세요).

**정말 iPad가 필요한가요?**
아니요. 보드는 평범한 웹 페이지입니다 — 아무 휴대폰, 태블릿, 브라우저에서나 여세요. 남는 iPad는 그저
항상 켜 두는 벽걸이 디스플레이로 쓰기 좋을 뿐입니다. 데스크톱 배너와 메뉴 바 요약도 함께 제공됩니다.

**제 코드나 데이터가 어딘가로 전송되나요?**
아니요 — 기본적으로 여러분의 에이전트에 관한 어떤 것도 여러분의 머신을 떠나지 않습니다. Andon은 완전히 셀프
호스팅됩니다: 계정도, 텔레메트리도, 분석도, "몰래 본사로 연결(phone home)"하는 일도 없습니다. 오직 대략적인 상태
정보(상태값, 프로젝트 이름, 한 줄 메시지)만 보관할 뿐 — 여러분의 코드나 로그, 시크릿은 절대 보관하지 않습니다.

솔직하게 밝히는 세 가지 단서: (1) 보드는 웹 폰트를 직접 셀프 호스팅하지 않는 한 Google Fonts에서 불러옵니다 — 이
요청에는 에이전트 데이터가 전혀 담기지 않으며, 그저 브라우저의 일반적인 폰트 요청일 뿐입니다. (2) 선택적 기능
(휴대폰 푸시와 호스팅 릴레이)은 **철저히 옵트인 방식**이며, 각각 여러분의 머신을 떠나는 것이 정확히 무엇인지
명시합니다 — 호스팅 릴레이는 *그조차도* 여러분 에이전트의 메시지를 읽을 수 없도록 설계되었습니다. 이 기능들은
로컬 우선이라는 이 기본값을 결코 바꾸지 않습니다. (3) 이 문서 사이트 자체는(Andon 도구가 아니라) 방문 수를
집계하기 위해 Cloudflare의 쿠키 없는 Web Analytics를 사용합니다 — 쿠키도, 사이트 간 추적도, 에이전트 데이터도
없습니다; 여러분이 실행하는 보드와 릴레이는 이를 절대 불러오지 않습니다.
