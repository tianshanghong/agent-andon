## 故障排查

- **看板设备打不开页面** —— 在同一个 Wi-Fi 吗？用的是 `http` 而不是 `https` 吗？你电脑的防火墙
  允许入站连接吗（macOS 上：系统设置 → 网络 → 防火墙）？IP 复制对了吗
  （它会在启动时打印出来，`andon doctor` 也会再打印一次）？
- **Claude 的 hook 毫无反应** —— 运行一次 `claude --debug`，留意 hook 报错；
  重新运行 `andon install claude`；再用 `andon doctor` 确认。
- **Codex 卡片从不出现 / 从不变化** —— 在 Codex 里运行一次 `/hooks` 来信任这些
  hook（Codex 会跳过不受信任的 hook）；`andon doctor` 可确认是否接好。
- **某张“工作中”卡片卡住不动** —— 很可能是某个进程在发送结束事件之前就挂掉了。
  它会在 6 小时后自动清除；对于 Codex，在该项目目录下运行 `andon post gone codex` 可立即清除它。
- **看板没有提示音** —— 点按一次 **开启提醒**（在你点击之前，浏览器会静音）；在
  手机上，看板必须通过 **HTTPS** 访问才能推送（参见 [运行 Andon](/zh-cn/docs/running/)）。

## 常见问题

**如何在 Claude Code 完成或需要批准时收到通知？**
运行 `andon serve`（桌面提醒默认开启）和 `andon install claude`。某个会话需要你或完成的那一刻，你就会
收到桌面横幅，外加任意设备上的实时看板。

**我可以同时监控多个 Claude Code / Codex 会话吗？**
可以——这正是它的意义所在。每个会话各占一行，需要你的那个会浮到最上面。

**它支持 OpenAI Codex 吗？**
支持。`andon install codex` 会接好 Codex 的生命周期 hook（运行一次 `/hooks` 来信任它们）。

**我真的需要一台 iPad 吗？**
不需要。看板就是一个普通网页——在任意手机、平板或浏览器上都能打开。一台闲置的 iPad 只是正好可以
当作一块不错的常开壁挂显示屏。你还会得到桌面横幅和菜单栏摘要。

**我的代码或数据会被发送到任何地方吗？**
不会——默认情况下，关于你 agent 的任何信息都不会离开你的机器。Andon 完全自托管：无账号、无遥测、
无访问统计、不"偷偷回传"。它只持有粗粒度的状态（状态、项目名、一行消息）——绝不包含你的代码、
日志或密钥。

三点诚实的提醒：(1) 除非你自托管字体，否则看板会从 Google Fonts 加载它的网页字体——这个请求不携带
任何 agent 数据，只是你浏览器正常的字体获取。(2) 可选功能（手机推送，以及托管中继）**严格按需主动开启**，
且每一项都会明确说明哪些东西会离开你的机器——托管中继的设计使得即便是*它*也读不到你 agent 的消息。
它们绝不会改变这个本地优先的默认设置。(3) 这个文档站点本身（而非 Andon 工具）使用 Cloudflare 的无
cookie Web Analytics 来统计访问量——无 cookie、无跨站追踪、无 agent 数据；你运行的看板和中继都从不加载它。
